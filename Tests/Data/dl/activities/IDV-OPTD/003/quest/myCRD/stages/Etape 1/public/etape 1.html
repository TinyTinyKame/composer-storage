Etape 1 - Réalisation du projet

Le programme doit implémenter la totalité des opérations nécessaires pour exécuter les commandes qui lui sont passées :

  • Ajouter une entrée
  • Supprimer une entrée
  • Rechercher une entrée.

Le programme ne prend aucun argument. Son fichier de commandes est lu sur l'entrée standard.

Les réponses sont fournies sur la sortie standard.

Vous n'avez pas à stocker les valeurs entre deux exécutions.

En fonction des commandes passées, les réponses diffèrent.

  • Commande de création :
    • si la clée existe, la valeur est mise à jour.
    • si la clée n'existe pas, la paire clée/valeur est ajoutée.
    • dans les deux cas, la clée est retournée.
  • Commande de suppression :
    • si la clée existe, sa valeur est retournée.
    • si la clée n'existe pas, -1 est retourné.
  • Commande de recherche :
    • si la clée existe, sa valeur est retournée.
    • si la clée n'existe pas, -1 est retourné.

Format des commandes

Le format de commande permet de gérer trois types de commandes :

  • création;
  • suppression;
  • recherche.

Format d'une commande de création :

‹clée›‹espace›‹valeur›\n

Format d'une commande de suppression :

‹clée›‹espace›D\n

Format d'une commande de recherche :

‹clée›\n

Exemple de fichier de commande :

32 32 D 34 4 34 34 5 34 34 D 34

Sortie attendue :

-1
-1
34
4
34
5
5
-1
